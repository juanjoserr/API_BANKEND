🚀 PRÓXIMOS PASOS
═══════════════════════════════════════════════════════════════════════════════

1️⃣  CONFIGURAR BASE DE DATOS

    Opción A: db4free.net (Recomendado)
    ─────────────────────────────────
    a) Regístrate en https://www.db4free.net/
    b) Crea una base de datos
    c) Edita .env con tus credenciales:
    
       DB_HOST=db4free.net
       DB_USER=tu_usuario
       DB_PASSWORD=tu_contraseña
       DB_NAME=tu_base_datos

    Opción B: MySQL Local
    ─────────────────────────────────
    DB_HOST=localhost
    DB_USER=root
    DB_PASSWORD=tu_contraseña_local
    DB_NAME=api_backend


2️⃣  CAMBIAR JWT_SECRET (IMPORTANTE)

    Edita .env y reemplaza:
    
    JWT_SECRET=your_super_secret_jwt_key_change_this_in_production...
    
    Con una clave aleatoria fuerte (mínimo 32 caracteres):
    
    Ejemplos para generar:
    • Linux/Mac: openssl rand -base64 32
    • En línea: https://www.uuidgenerator.net/ (copia varias veces)


3️⃣  INICIAR SERVIDOR

    Opción A: Desarrollo (con auto-reload)
    ──────────────────────────────────────
    npm run dev

    Opción B: Producción
    ──────────────────────────────────────
    npm start

    El servidor estará disponible en: http://localhost:3000


4️⃣  PROBAR LA API

    a) Abre http://localhost:3000 en el navegador
    
    b) Usa Postman o curl para probar endpoints:
    
       REGISTRO:
       POST http://localhost:3000/auth/register
       Body: { "email": "test@example.com", "password": "123456", "confirmPassword": "123456" }
       
       LOGIN:
       POST http://localhost:3000/auth/login
       Body: { "email": "test@example.com", "password": "123456" }
       
       CREAR ITEM (necesita token):
       POST http://localhost:3000/api/items
       Headers: Authorization: Bearer {token_del_login}
       Body: { "nombre": "Mi item", "descripcion": "Descripción", "estado": "activo" }


5️⃣  USAR DOCKER (OPCIONAL)

    Construir imagen:
    docker build -t api-backend .

    Ejecutar contenedor:
    docker run -p 3000:3000 --env-file .env api-backend

    O con Docker Compose:
    docker-compose up -d


6️⃣  HACER PUSH A GIT (OPCIONAL)

    git remote add origin https://github.com/tu_usuario/api-backend.git
    git branch -M main
    git push -u origin main


📊 ENDPOINTS DISPONIBLES
═══════════════════════════════════════════════════════════════════════════════

AUTENTICACIÓN (Sin token requerido):
  POST   /auth/register        Registrar nuevo usuario
  POST   /auth/login           Iniciar sesión (obtiene token)

ITEMS (Requiere token JWT en header Authorization):
  GET    /api/items            Obtener todos los items
  GET    /api/items/:id        Obtener item específico
  POST   /api/items            Crear nuevo item
  PUT    /api/items/:id        Actualizar item
  DELETE /api/items/:id        Eliminar item

PRUEBA:
  GET    /                     Verificar que servidor está corriendo

🆘 SOLUCIÓN DE PROBLEMAS
═══════════════════════════════════════════════════════════════════════════════

❌ Error: "Cannot connect to database"
   → Verifica credenciales en .env
   → Confirma que BD esté en línea
   → Prueba con: mysql -h {host} -u {user} -p

❌ Error: "Port 3000 already in use"
   → Cambia PORT en .env
   → O termina proceso: lsof -i :3000 | kill -9

❌ Error: "Invalid token"
   → Asegúrate de enviar en header: Authorization: Bearer {token}
   → Token puede haber expirado (24 horas)
   → Verifica que JWT_SECRET sea igual en .env

❌ Error: "npm install falla"
   → Elimina node_modules: rm -rf node_modules
   → Limpia cache: npm cache clean --force
   → Reinstala: npm install

❌ Docker error: "Cannot connect to Docker daemon"
   → Inicia Docker Desktop
   → O usa: sudo systemctl start docker (Linux)
