🚀 CÓMO EMPEZAR
═══════════════════════════════════════════════════════════════════════════════

PASO 1: Configurar Base de Datos
────────────────────────────────
Edita el archivo .env con tus credenciales:

  DB_HOST=db4free.net         (o tu servidor MySQL)
  DB_USER=tu_usuario
  DB_PASSWORD=tu_contraseña
  DB_NAME=tu_base_datos
  DB_PORT=3306

PASO 2: Cambiar JWT_SECRET
────────────────────────────
Reemplaza en .env:

  JWT_SECRET=genera_una_clave_aleatoria_fuerte

PASO 3: Iniciar Servidor
────────────────────────────
Opción A - Desarrollo:
  npm run dev

Opción B - Producción:
  npm start

PASO 4: Probar API
────────────────────────────
Accede a: http://localhost:3000

Usa Postman o curl para probar endpoints


🐳 USAR CON DOCKER
═══════════════════════════════════════════════════════════════════════════════

Compilar imagen:
  npm run docker:build
  # o
  docker build -t api-backend .

Ejecutar contenedor:
  npm run docker:run
  # o
  docker run -p 3000:3000 --env-file .env api-backend

Con Docker Compose:
  npm run docker:compose
  # o
  docker-compose up -d

Detener:
  npm run docker:compose:down
  # o
  docker-compose down

📞 SOPORTE Y AYUDA
═══════════════════════════════════════════════════════════════════════════════

Problema: BD no conecta
Solución: Verifica .env, credenciales y que BD esté disponible

Problema: Puerto 3000 en uso
Solución: Cambia PORT en .env o termina proceso en ese puerto

Problema: Token expirado
Solución: Haz login nuevamente para obtener token nuevo

Problema: Docker no funciona
Solución: Asegúrate Docker esté instalado e iniciado

Problema: npm install falla
Solución: Limpia cache (npm cache clean) y reinstala
